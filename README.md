## Getting Started

First, run the development server:

```bash
npm run dev
# or
yarn dev
# or
pnpm dev
# or
bun dev
```

Open [http://localhost:3000](http://localhost:3000) with your browser to see the result.

You can start editing the page by modifying `app/page.tsx`. The page auto-updates as you edit the file.

This project uses [`next/font`](https://nextjs.org/docs/app/building-your-application/optimizing/fonts) to automatically optimize and load [Geist](https://vercel.com/font), a new font family for Vercel.

Worker build & deploy

1. Build workers bundle:
   - `node scripts/buildWorkers.js` (requires TypeScript)
2. Build a worker image using Dockerfile.worker and push to registry.
3. Deploy the k8s manifest in k8s/worker-deployment.yaml (update image name).

Metrics

- Worker exposes Prometheus metrics at /api/metrics (when the web server is running in the same container). If you run the worker process separately, consider adding a small HTTP server to expose metrics or run a sidecar.

Systemd

- Example systemd service is in scripts/systemd/gc-worker.service. Update paths and environment variables as needed and enable with `systemctl enable --now gc-worker`.
